1.  Problem Statement 
 
Traditional approaches to academic performance evaluation rely heavily on post-exam results, 
leaving limited time for meaningful interventions. By the time a student's grade is known, opportunities 
to course-correct have often passed. This project addresses a critical gap: the absence of an early-
warning, data-driven system that can predict a student's academic grade before any examinations 
are taken. 
 
The Student Grade Predictor  is designed to analyze underlying behavioral traits — including study 
habits, attendance records, class participation, reading frequency, and demographic factors — to 
proactively identify at-risk students. The system provides actionable foresight rather than a 
retrospective summary, enabling teachers, counselors, and parents to intervene when it matters 
most. 
 
Key challenges addressed by this project include: 
• Multi-class classification across five grade categories (A, B, C, D, F) with significant class 
imbalance 
• Feature engineering from behavioral signals that lack direct numerical relationships with 
academic outcomes 
• Achieving high sensitivity for Grade F (at-risk) predictions to ensure no struggling student 
goes undetected 
• Deploying the model as an interactive Streamlit web application with real-time inference 
 
2.  Objective of the Study 
 
The primary objectives of this project are: 
• To build a predictive ML pipeline that classifies student grades (A/B/C/D/F) using only 
behavioral and demographic inputs — without relying on exam scores as input features 
• To engineer meaningful interaction features (Engagement_Index, Study_Efficiency, 
Study_Attendance_Index) that capture compound behavioral effects 
• To compare multiple machine learning models — XGBoost, Random Forest, and Logistic 
Regression — on their ability to classify academic outcomes 
• To maximize F1-score for Grade F to ensure early detection of at-risk students 
• To develop a modular, production-ready Streamlit application that provides real-time grade 
predictions, confidence scores, feature importance, and personalized intervention 
recommendations 
• To cluster students into behavioral personas using K-Means for holistic profiling

4.  Data Preparation & Processing 
 
Before model training, the dataset underwent a rigorous preprocessing pipeline to ensure data quality, 
consistency, and readiness for machine learning algorithms. 
 
4.1  Preprocessing Steps 
• Dataset Loading: Loaded the 200,000-row CSV dataset with 21 columns using pandas, 
verifying schema integrity and data types 
• Duplicate Removal: Identified and removed any duplicate Student_ID entries to prevent data 
leakage 
• Missing Value Handling: Imputed missing numeric values using column medians; categorical 
nulls filled with mode values 
• Data Type Conversion: Ensured numeric columns (Age, Study_Hours_Per_Day, etc.) were 
cast to appropriate float/int types 
• Categorical Encoding: Applied One-Hot Encoding to categorical features (Gender, 
High_School_Type, Parental_Education, etc.) within the scikit-learn pipeline 
• Feature Scaling: Applied StandardScaler to numeric features within the pipeline 
preprocessor 
• Train/Test Split: Stratified 80/20 split preserving grade class distribution

6.  Models Used 
 
Three machine learning models were trained and evaluated, each representing a distinct algorithmic 
approach to the multi-class classification problem. 
 
6.1  Logistic Regression (Baseline) 
Logistic Regression served as the statistical baseline model. A multi-class Softmax formulation was 
applied with L2 regularization. Despite its computational efficiency, Logistic Regression struggled to 
capture the non-linear relationships inherent in behavioral data, resulting in significantly lower 
accuracy. 
• Type: Linear probabilistic classifier with One-vs-Rest strategy 
• Purpose: Establish a performance lower-bound for comparison 
• Pipeline: StandardScaler → OneHotEncoder → LogisticRegression 
 
6.2  Random Forest (Ensemble Method) 
Random Forest is an ensemble of decision trees trained using bootstrap aggregation (Bagging). Each 
tree is trained on a random subset of features and data, and predictions are aggregated by majority 
vote. This approach naturally handles non-linearity and feature interactions. 
• Estimators: 200 trees 
• Max Depth: 15 
• Hyperparameter Tuning: RandomizedSearchCV with 5-fold cross-validation 
• Feature Selection: SelectKBest within pipeline for dimensionality efficiency 
 
6.3  XGBoost (Gradient Boosting) 
XGBoost (eXtreme Gradient Boosting) is a sequential ensemble method that builds trees iteratively, 
with each new tree correcting the residual errors of the previous ensemble. It incorporates L1/L2 
regularization, early stopping, and parallel tree construction. 
• Objective: multi:softmax for multi-class classification 
• Early Stopping: Enforced to prevent overfitting 
• Label Encoding: Applied via LabelEncoder to satisfy XGBoost's integer label requirement 
• Hyperparameter Tuning: RandomizedSearchCV for n_estimators, max_depth, 
learning_rate, subsample 
 
6.4  K-Means Clustering (Persona Analysis) 
In addition to supervised classification, K-Means clustering (k=4) was applied on academic score 
features to group students into behavioral personas: The Consistent Achiever, The Science 
Specialist, The Dedicated Studier, and The At-Risk Student. This unsupervised component provides 
holistic student profiling beyond a single grade prediction.

7.2  Validation Strategy 
Multiple validation techniques were applied to ensure robust performance estimation: 
• Hold-Out Method: Standard 80/20 stratified train/test split 
• K-Fold Cross Validation (k=5): Applied to all models; macro F1 reported as primary CV 
metric 
• K-Fold with IMDb Weights: Custom weighted CV accounting for class imbalance 
• Leave-One-Out (LOO): Applied on a representative sample for high-variance estimation 
• Grid Search CV: Exhaustive hyperparameter search on critical parameters

11.  Prediction & Intervention System 
 
The application provides three complementary outputs for each student prediction: 
 
11.1  Grade Prediction 
The selected model (XGBoost, Random Forest, or Logistic Regression) produces a predicted grade 
(A–F) along with a full probability distribution across all five classes. This confidence vector is 
displayed as a bar chart, allowing educators to assess prediction certainty. 
 
11.2  Student Persona (K-Means) 
When academic score inputs are provided, K-Means assigns the student to one of four behavioral 
clusters: 
• Cluster 0 — The Consistent Achiever: Balanced performance across all dimensions 
• Cluster 1 — The Science Specialist: STEM-strong with humanities gaps 
• Cluster 2 — The Dedicated Studier: High effort, high note-taking, disciplined 
• Cluster 3 — The At-Risk Student: Low attendance and engagement flags 
 
11.3  Intervention Recommendations 
A rule-based intervention engine (utils.py) generates personalized, data-driven recommendations 
based on the predicted grade and input feature values. Recommendations target attendance 
improvement, study schedule restructuring, participation goals, reading habit building, and note-
taking strategies. For Grade F predictions, an immediate escalation protocol is triggered.

13.  Conclusion 
 
The Student Grade Predictor  successfully demonstrates that behavioral and demographic data, 
when properly engineered and modeled, can accurately predict academic outcomes long before 
examinations take place. With XGBoost achieving 94.77% test accuracy and a 97.25% F1-score for 
Grade F identification, the system provides robust early-warning capability that can meaningfully 
support educational interventions. 
 
The project validates the core hypothesis: that a predictive ML system analyzing behavioral traits 
creates actionable foresight rather than retrospective reporting. The modular Streamlit application 
bridges the gap between model performance and real-world utility, enabling teachers and counselors 
to query student predictions in real time. 
 
The comparative analysis confirms that tree-based ensemble methods (XGBoost, Random Forest) 
significantly outperform linear models (Logistic Regression) for this problem domain, and that 
interaction feature engineering is a high-value investment, with engineered features appearing in the 
top-5 importance rankings. The addition of K-Means persona clustering enriches the prediction with 
a holistic behavioral profile, providing a richer tool for educators than a single grade label alone. 
 
Key Insights: 
• Behavioral signals alone — without any exam scores — are sufficient to predict student 
grades with >94% accuracy, demonstrating the predictive power of engagement and habit 
data 
• XGBoost and Random Forest are statistically equivalent on this dataset; XGBoost edges 
ahead in Grade F detection (97.25% F1 vs 96.98%), making it the recommended production 
model 
• Logistic Regression fails on this task (47.61% accuracy), confirming the non-linearity of 
behavioral–grade relationships 
• Attendance Percentage is the single most important feature, contributing ~25% of total 
model importance 
• The Engagement_Index interaction feature, computed as (Attendance × 
Class_Participation) / 100, is the highest-ranked engineered feature and validates the 
interaction feature engineering approach 
• The model is particularly vigilant for at-risk students (Grade F: 97.25% F1), which is the 
most operationally important outcome for early intervention programs 
• K-Means persona clustering reveals that ~25% of students fall into the At-Risk cluster, 
enabling batch identification for counseling outreach
